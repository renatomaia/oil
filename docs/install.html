Download

You can download the OiL package from the following link. See below for instructions about building OiL and its required software packages.

  • OiL 0.4 beta [tar.gz] [zip]

Windows Binaries: If you just want to give OiL a try in a Windows platform then just download the OiL Win32 Binaries and extract its contents into a folder. After this, you will be able to run all the demos included in the package by executing the batch files included in the folder of each demo.

Building

If you want to use OiL in a Unix system or want to make a particular OiL built in order to include it in an application, then you have to build some required software packages, namely the Lua language and the libraries for socket and bit manipulation support in Lua. The required software packages are listed below.

  • Lua 5.1
  • LuaSocket 2.0
  • OiL's Bit Manipulation Library (included in the OiL package)

Note: If you plan to build any of the packages listed here in a Windows platform and you don't have a C compiler, then you can donwload for free the Microsoft Visual Studio C++ Express and Microsoft Platform SDK. Follow the instructions defined here.

Required Software Packages

Lua 5.1

Download Lua 5.1 package from Lua web site, then follow the instructions in INSTALL file included in the package. If you plan to build Lua in a Windows platform, then you can use the Lua 5.1 package provided by the OiL team. To build the package provided by OiL team you must do the following:

Linux:
  1. go to the folder you extracted the package files
  2. read and edit file Makefile to suit your needs
  3. run command make linux install
Windows: (dynamic libraries).
  1. go to the folder you extracted the package files
  2. read and edit file Makefile.win32 to suit your needs
  3. run command nmake /f Makefile.win32 dll install
Windows: (static libraries)
  1. go to the folder you extracted the package files
  2. read and edit file Makefile.win32 to suit your needs
  3. run command nmake /f Makefile.win32 static install

LuaSocket 2.0

Download LuaSocket 2.0 package from LuaSocket web site, then follow the instructions in INSTALL file included in the package. If you have any problems to use the LuaSocket official package, you can try the LuaSocket 2.0 package provided by the OiL team. To build the package provided by OiL team you must do the following:

Linux:
  1. go to the folder you extracted the package files
  2. read and edit file config to suit your needs
  3. run command make linux install
Windows: (dynamic libraries that can be loaded from Lua console).
  1. go to the folder you extracted the package files
  2. read and edit file config.win32 to suit your needs
  3. run command nmake /f Makefile.win32 dll install
Windows: (static libraries)
  1. go to the folder you extracted the package files
  2. read and edit file config.win32 to suit your needs
  3. run command nmake /f Makefile.win32 static install-lib

OiL 0.4

OiL is almost entirely written in Lua, however it uses a bit manipulation library written in C. Therefore, to install OiL, you need to build its bit manipulation support. To build and install OiL you must choose one of the options described below.

Standard

Installs the Lua files and bit manipuation library for dynamic loading. This option is useful for playing with OiL implementation, because you will be able to modify its implementation by changing the Lua files.

Unix:
  1. go to the folder you opened the package
  2. read and edit file config to suit your needs
  3. run command make install
Windows:
  1. go to the folder you opened the package
  2. read and edit file config.win32 to suit your needs
  3. run command nmake /f Makefile.win32 install

Pre-Compiled

Installs a set of libraries containing all modules that compose OiL implementation already pre-compiled. This option is useful for users that don't intend to make changes in OiL implemenation.

Unix:
  1. go to the folder you opened the package
  2. read and edit file config to suit your needs
  3. run command make installb
Windows:
  1. go to the folder you opened the package
  2. read and edit file config.win32 to suit your needs
  3. run command nmake /f Makefile.win32 installb

Preloading

Installs a single library containing the bit manipuation library and all OiL Lua files already pre-compiled for preloading into embedded Lua applications. This option is useful for developers that intend to include OiL in other applications by linking into a single file. See demo 'run' for more details.

Unix:
  1. go to the folder you opened the package
  2. read and edit file config to suit your needs
  3. run command make installp
Windows:
  1. go to the folder you opened the package
  2. read and edit file config.win32 to suit your needs
  3. run command nmake /f Makefile.win32 installp

Environment Setup

In order to load OiL package in the Lua stand-alone interpreter, you must add the paths where you installed OiL and LuaSocket to the LUA_PATH and LUA_CPATH environment variables. For example, if you installed Lua files of OiL and LuaSocket (see INSTALL_LMOD macro in file config) into directory /usr/local/share/lua/5.1/ then LUA_PATH must be set to:

LUA_PATH="/usr/local/share/lua/5.1/?.lua;/usr/local/share/lua/5.1/?/init.lua"

A similar approach is used for the LUA_CPATH environment variable. Supose you have installed the C libraries of OiL and LuaSocket (see INSTALL_CMOD macro in file config) into directory /usr/local/lib/lua/5.1/ then LUA_CPATH must be set to:

LUA_CPATH="/usr/local/lib/lua/5.1/?.so"

For further information on the use of LUA_PATH and LUA_CPATH environment variable, check the Lua manual.
